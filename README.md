Repositório referente as etapas do curso Azure.
Este README tem como objetivo mostrar o processo de criação do ambiente básico, que foi apresentado até este momento no curso Microsoft AI for Tech - Azure Databricks.

➡️ Definição de tipos de serviços cloud:

 Existem 3 modelos de serviçõs cloud.

◾ IaaS - Infrastructure as a Service.
 - Modelo em nuvem que oferece servidores, armazenamento e rede sobre onde não existe a nescessidade de hardware físico.
 - O acesso a esse tipo de serviço ocorre por meio de uma interface web ou API, por meio de recursos virtualizados e hospedados em Data Centers
   de provedores de nuvens.

◾ PaaS - Plataform as a Service.
 - Modelo em nuvem onde o provedor do serviço fica responsável pela manutenção, escalabilidade e infraestrutura; o clinte fica com a parte do desenvolvimento:
   compilar, implatar e gerenciar aplicativos.

◾ SaaS - Software as a Service.
 - Modelo em nuvem onde o provedor entrega um software baseado em nuvem e o cliente pode acessar os aplicativos pela internet. Este modelo opera com base
    em assinaturas, elimina a nescessidade de instalação ou manutenção local do software.
    Exemplos comuns de Saas:
    ▪️Serviços de emails - Gmail e Outlook.com
    ▪️ Armazenamento online - Dropbox e Google Drive
    ▪️APPs de produtividade - Trello e Notion

➡️ Tipos de Clouds:

◾ Pública
 - É um modelo onde o cliente usa por meio da internet os recursos de um provedor (como a Azure) em vez de servidores físicos no local.
   
◾ Privada
 - É um modelo de computação em nuvem onde os recursos são dedicados exclusivamente a uma única organização, onde os dados e a infraestrutura pertence somente a ela (hardware e software). Garantindo maior controle, segurança e personalização.

◾ Híbrida
 - É um modelo onde que combina a nuvem privada ou infraestrutura local (on-premises) com uma nuvem pública (Azure) que se integram.
